# i18n

All visible strings in the graphic editor go through `I18nService` from `@core/i18n`. This page lists every `graphic.*` key and explains how to add a locale.

## Where the strings live

| File | Purpose |
| ---- | ------- |
| `packages/core/src/i18n/locales/en.ts` | English (default) |
| `packages/core/src/i18n/locales/uk.ts` | Ukrainian |
| `packages/graphic-editor/src/i18n/keys.ts` | Typed constants (avoid magic strings in code) |

## Adding a locale

1. Copy `packages/core/src/i18n/locales/en.ts` to a new file, e.g. `fr.ts`.
2. Translate every string value.
3. Export the constant as `fr` and register it in `packages/core/src/i18n/i18n.ts` (follow the existing `uk` pattern).
4. Pass `locale: 'fr'` to `GraphicEditor.init()`.
5. Run `npm run check:graphic-i18n` — it checks `en` and `uk` parity; you may want to extend it for your locale.

## Audit script

`npm run check:graphic-i18n` (`scripts/check-graphic-i18n.mjs`) verifies that:

- Every `graphic.*` key referenced in code exists in `en.ts` and `uk.ts`.
- No `graphic.*` key in `en.ts` is unused (no orphans).

Run it before each release or wire it into CI.
