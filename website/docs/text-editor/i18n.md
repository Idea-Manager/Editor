# Text editor: i18n

The editor’s `I18nService` (from `@core`) loads **English** and **Ukrainian** dictionaries from the core package. For custom blocks, toolbars, and plugins, use **`TextEditorOptions.i18nOverrides`**: a flat object of string keys that is **merged on top of** the active locale’s file.

## API

- **`import { I18nService, mergeDictionaries, type TranslationDictionary } from '@core';`**
- **`TextEditorOptions.i18nOverrides?: Partial<TranslationDictionary>`** — passed to `I18nService` in `TextEditor.init()`. Keys you define override or extend the base dictionary. When merging, entries with an `undefined` value are skipped.

`mergeDictionaries` is available if you need the same merge behavior outside the editor (e.g. app-level helpers).

## Key naming for extensions

- Use a **namespace prefix** so you never collide with core or other plugins, for example: `myProduct.blocks.callout`, `acme.plugin.toolbarBold`.
- Block palette and slash items use `labelKey` on the block; floating toolbar extra buttons use `titleKey` and `labelKey` as defined in the toolbar config. Those keys must exist in the merged dictionary, either in core locales or in `i18nOverrides`.

## Built-in keys used by `@text-editor` (reference)

These are defined in [`packages/core/src/i18n/locales/en.ts`](https://github.com/Idea-Manager/Editor/blob/master/packages/core/src/i18n/locales/en.ts) (and `uk.ts`). You can override any of them via `i18nOverrides` if you need different wording in the text editor.

| Area | Sample keys (not exhaustive) |
| ---- | ----------------------------- |
| Blocks | `block.paragraph`, `block.heading`, `block.listOrdered`, `block.listUnordered`, `block.table`, `block.embed` |
| Editor | `editor.placeholder` |
| Floating toolbar | `toolbar.alignLeft`, `toolbar.alignCenter`, `toolbar.alignRight`, `toolbar.textColor`, `toolbar.addLink`, `toolbar.linkUrlPlaceholder` |
| Gutter | `gutter.addBlock`, `gutter.dragToReorder`, `gutter.removeBlock`, `gutter.confirmRemoveTitle`, `gutter.confirmRemoveMessage`, `gutter.modalCancel`, `gutter.modalConfirm` |
| Slash palette | `slash.noResults` |
| Table (menus + size picker) | `table.insertRowAbove`, `table.*`, `table.createTableTitle`, `table.borders`, `table.cancel`, `table.create`, … (see `en.ts`) |
| Embed | `embed.placeholder`, `embed.remove`, `embed.open` |
| Color picker (table UI) | `colorPicker.custom`, `colorPicker.select`, `colorPicker.cancel` |

The host application may use many more keys (top bar, import/export, shortcuts, status bar) from the same `en` file; the text editor instance only **requires** the keys above for built-in editor UI. Custom blocks and toolbar entries should document their own `labelKey` / `titleKey` values for integrators.

## Multiple editors and locales

The editor’s `I18nService` is **per** `TextEditor` instance, created in `init()` from `locale` and `i18nOverrides`. Different instances can use different overrides.

See [Theming](./theming.md) for how **global** default CSS is shared when several editors are on the same page.
